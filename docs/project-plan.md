# 拼省省商城 10 天项目计划

## 一、交付目标

完成一个可以真实启动并演示核心业务链路的电商微服务项目，并形成：

- 可访问的 Git 仓库
- 可复现的启动文档
- 核心接口和业务流程说明
- 运行截图、测试记录和问题复盘
- 可直接改写到简历中的项目描述

## 二、10 天安排

| 天数 | 工作内容 | 验收结果 |
|---|---|---|
| 1 | 初始化仓库，确认模板、版本、模块和启动策略 | Git 仓库可提交，项目范围明确 |
| 2 | 准备 MySQL、Redis、Nacos、RabbitMQ 等基础服务 | 基础服务端口和控制台可访问 |
| 3 | 导入微服务骨架，启动 Gateway、Auth、System | 登录和后台管理链路可用 |
| 4 | 完成商品数据和商品详情接口 | 能按商品或 SKU 查询详情 |
| 5 | 加入 Redis 缓存、延时双删、分布式锁和缓存防护 | 缓存命中、失效和并发场景可验证 |
| 6 | 完成购物车增删改查、选中和全选 | 购物车接口和页面可联调 |
| 7 | 完成地址、结算和订单创建 | 可以生成订单和订单明细 |
| 8 | 完成库存锁定、RabbitMQ 消息和延迟处理 | 下单异常时库存可释放或补偿 |
| 9 | 完成支付状态流转和前后端联调 | 模拟支付闭环可演示 |
| 10 | 测试、修复、文档、截图、Git 整理和简历材料 | 能从零按 README 启动并演示 |

## 三、实现优先级

### 必须完成

- 用户登录
- 商品详情
- Redis 缓存
- 购物车
- 订单创建
- 库存锁定
- RabbitMQ 基本消息流程
- 模拟支付
- 前后端联调

### 有余力再完成

- 支付宝沙箱支付
- MinIO 图片上传
- Sentinel 限流
- Docker Compose 一键启动
- 接口自动化测试和简单压测

## 四、每日工作闭环

```text
阅读业务需求
    ↓
确认模块和数据流
    ↓
实现一个小功能
    ↓
启动并调用真实接口
    ↓
记录测试结果
    ↓
提交一次有意义的 Git commit
```

## 五、简历验收标准

- 不写没有实际验证过的功能
- 不虚构性能指标，性能数据必须来自测试记录
- 能解释每个微服务的职责
- 能画出商品、购物车、订单和库存之间的调用关系
- 能说明缓存一致性、分布式锁和消息可靠性分别解决什么问题
- README 能让别人知道依赖、端口、启动顺序和测试接口
